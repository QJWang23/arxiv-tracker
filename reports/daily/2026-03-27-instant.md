## PackForcing: Short Video Training Suffices for Long Video Sampling and Long Context Inference

**来源**: arxiv | **日期**: 2026-03-26 | **热度**: 20 | **标签**: inference-opt, long-context, hw-accel

### 核心技术要点
- **三分区 KV-cache 策略**：将历史上下文分为 Sink tokens（保留早期锚帧全分辨率）、Mid tokens（32x 压缩）和 Recent tokens（全分辨率保证局部时序连贯性）
- **动态 top-k 上下文选择**：对 Mid tokens 进行选择性保留，严格控制内存占用
- **连续 Temporal RoPE Adjustment**：无缝重新对齐因 token 丢弃造成的位置间隙

### 创新点
突破了自回归视频扩散模型的三大瓶颈：线性 KV-cache 增长、时序重复、累积误差。核心创新在于用 5 秒短视频训练即可实现 120 秒（24x）时序外推，KV-cache 仅需 4GB，单张 H200 即可生成 2 分钟 832x480@16fps 视频。

### 对关注领域的影响
- **推理优化**: 三分区 KV-cache 策略可借鉴到 LLM 长上下文推理，实现有界的内存占用
- **K8s 基础设施**: 单 GPU 长视频生成能力降低了分布式推理的复杂度，适合容器化部署
- **硬件加速**: 32x token 压缩通过 3D 卷积 + 低分辨率 VAE 实现，对 GPU 显存带宽友好

### 后续跟进
- [ ] 研究 Sink token 策略是否适用于 LLM 推理中的注意力稀疏化
- [ ] 对比 StreamingLLM 的 rolling cache 与 PackForcing 的三分区策略
- [ ] 评估在 K8s 上部署 PackForcing 的资源调度策略

## DFLOP: A Data-driven Framework for Multimodal LLM Training Pipeline Optimization

**来源**: arxiv | **日期**: 2026-03-26 | **热度**: 20 | **标签**: multimodal, hw-accel

### 核心技术要点
- **数据感知调度**：根据输入数据特征预测计算开销，动态平衡各 stage 和 microbatch 的负载
- **持续性能分析**：运行时捕获数据引起的计算方差
- **预测性调度**：将数据特征与执行计划耦合

### 创新点
首次指出分布式训练框架"数据盲"问题——异构多模态输入导致不同处理开销，造成 GPU 资源利用不均和同步延迟累积。DFLOP 通过数据驱动的流水线优化实现 **3.6x** 训练加速。

### 对关注领域的影响
- **推理优化**: 数据感知调度思想可迁移到推理服务的负载均衡
- **K8s 基础设施**: 可集成到 K8s 上的 PyTorch 分布式训练调度器
- **硬件加速**: 提升 GPU 利用率，减少空闲等待时间

### 后续跟进
- [ ] 研究是否可将数据感知调度应用于 vLLM/TGI 等推理框架
- [ ] 评估 DFLOP 与 DeepSpeed/Megatron 的兼容性

## ARMOR: Missing-Aware Multimodal Fusion for Unified Microservice Incident Management

**来源**: arxiv | **日期**: 2026-03-26 | **热度**: 20 | **标签**: multimodal

### 核心技术要点
- **模态特定非对称编码器**：隔离 metrics、logs、traces 的分布差异
- **缺失感知门控融合**：使用可学习占位符和动态偏差补偿防止不完整输入的跨模态干扰
- **自监督自回归 + 掩码引导重建**：联合优化异常检测(AD)、故障分类(FT)、根因定位(RCL)

### 创新点
针对微服务 AIOps 中网络波动和 agent 故障导致的数据缺失问题，设计了鲁棒的多模态融合框架。AD 和 RCL 不需要故障标签，仅 FT 需要故障类型标注。

### 对关注领域的影响
- **推理优化**: 缺失感知融合可应用于边缘计算场景下的部分传感器失效
- **K8s 基础设施**: 直接适用于 K8s 集群的自动化故障诊断，支持 metrics/logs/traces 三模态
- **硬件加速**: 自监督学习降低标注成本，适合大规模部署

### 后续跟进
- [ ] 在生产 K8s 集群中部署 ARMOR 进行 POC 验证
- [ ] 研究与 Prometheus/Grafana 的集成方案

## LanteRn: Latent Visual Structured Reasoning

**来源**: arxiv | **日期**: 2026-03-26 | **热度**: 20 | **标签**: multimodal

### 核心技术要点
- **连续视觉思维嵌入**：在推理过程中生成和关注 latent visual representations
- **两阶段训练**：SFT 建立 visual features 与 latent states 的关联，RL 对齐 latent reasoning 与任务效用
- **直接在 latent space 推理**：避免将视觉内容 verbalize 为文本

### 创新点
突破了 LMM 依赖文本化视觉内容的限制，实现了语言与紧凑潜在视觉表示的交替使用。在 VisCoT、V*、Blink 三个感知基准上取得一致提升。

### 对关注领域的影响
- **推理优化**: Latent space 推理比 pixel space 更高效，减少计算开销
- **硬件加速**: 紧凑的潜在表示降低内存带宽压力

### 后续跟进
- [ ] 对比 LanteRn 与 CoT (Chain-of-Thought) 的计算效率
- [ ] 研究是否可将 latent visual reasoning 应用于视频理解任务
